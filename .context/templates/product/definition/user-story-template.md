## Examples

### Example 1: Document Search

```
Como desarrollador,
quiero buscar documentos técnicos por contenido semántico,
para encontrar rápidamente información relevante sin conocer palabras exactas.
```

**Acceptance Criteria:**
- [ ] User can enter natural language queries
- [ ] Results are ranked by semantic relevance
- [ ] Search works across all document types
- [ ] Results include document snippets with highlighted matches
- [ ] Search response time < 500ms

### Example 2: Document Relationships

```
Como tech lead,
quiero ver visualmente cómo se relacionan los documentos técnicos,
para entender el impacto de cambios y decisiones.
```

**Acceptance Criteria:**
- [ ] Interactive graph shows document relationships
- [ ] Different relationship types are visually distinguished
- [ ] User can click nodes to view document details
- [ ] Graph updates in real-time as documents change
- [ ] Performance acceptable with 1000+ documents
